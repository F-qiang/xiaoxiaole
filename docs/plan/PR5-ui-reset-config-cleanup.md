# PR5（UI、重置、配置与工程化收尾）

## 目标
完成顶部信息区、结算弹窗、重置、配置管理和收尾测试，让项目具备 Demo 发布能力。

## 任务 18：实现顶部信息区
- **目标**：显示剩余障碍物数量和步数。
- **实现内容**：
  - 顶部状态栏
  - 步数实时更新
  - 障碍物数量实时更新
- **验收方式**：
  - 数值变化与游戏状态同步

## 任务 19：实现结算弹窗
- **目标**：展示胜负结果并支持重新开始。
- **实现内容**：
  - 胜负结果展示
  - 清除障碍物数量展示
  - 总步数展示
  - 重新开始按钮
- **验收方式**：
  - 结算界面正确出现
  - 点击重新开始能重置关卡

## 任务 20：实现重置逻辑
- **目标**：一键恢复到初始关卡状态。
- **实现内容**：
  - 清空当前棋盘
  - 重建初始配置
  - 步数清零
  - 障碍物数量恢复初始值
- **验收方式**：
  - 重置后和初始局完全一致
  - 可反复测试

## 任务 21：关卡配置文件化
- **目标**：用 JSON 配置文件管理 Demo 关卡数据，降低工程复杂度。
- **实现内容**：
  - 使用 JSON 管理关卡障碍物布局
  - 使用 JSON 管理棋子颜色配置
  - 使用 JSON 管理初始参数
  - 保持配置可读、可改、可快速替换
- **实现说明**：
  - 对于一测 Demo 来说，SQLite 属于过度设计：只有 1 个固定关卡时，JSON 配置文件已经完全足够
  - JSON 的工程复杂度更低，能减少数据库接入、封装和维护成本
  - 后续若需要多关卡、存档或复杂查询，再升级到 SQLite 即可
- **验收方式**：
  - 可通过修改 JSON 调整关卡
  - 不改代码也能修改障碍物布局与基础参数

## 任务 22：补齐表现层和资源管理
- **目标**：完成基础 UI、动画和资源加载。
- **实现内容**：
  - 棋子资源
  - 特效资源
  - 音效资源
  - 节点生命周期管理
- **验收方式**：
  - 游戏运行稳定
  - 资源加载与释放正常

## PR5 联调验证
- **完整对局测试**：连续跑 10 局验证稳定性。
- **重置与配置验证**：修改 JSON 后重新加载，关卡配置变化生效。
- **界面与资源稳定性验证**：结算、重置、资源加载和销毁均正常。
