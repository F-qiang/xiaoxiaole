# PR4（特效、障碍物与结算逻辑）

## 目标
完成障碍物清除、关卡目标与胜负判定的基础闭环，并为后续特效棋子与连锁规则预留扩展点。

## 任务 13：实现障碍物清除
- **目标**：普通三消可清除障碍物，障碍物清空后视为通关目标完成。
- **实现内容**：
  - 普通消除与障碍物重叠时，障碍物也一并清除
  - 障碍物剩余数量同步减少
  - 关卡目标显示为当前剩余障碍物数量
- **验收方式**：
  - 普通消除后障碍物可被正确清除
  - 剩余障碍物数量实时更新
  - 障碍物清零后可进入通关判定

## 任务 14：实现胜负判定
- **目标**：障碍物全部清空时胜利；盘面无合法交换时失败。
- **实现内容**：
  - 障碍物为 0 时胜利
  - 步数仅用于记录，不参与胜负判定
  - 盘面稳定且无可用交换时失败
  - 结算弹窗展示结果
- **验收方式**：
  - 胜利和失败都能正确触发
  - 胜利条件与障碍物清空保持一致
  - 步数不会误判为失败条件

## PR4 联调验证
- **边界场景测试**：覆盖障碍物清除、无合法交换、操作锁定等边界 Case。
- **规则验证**：检查障碍物清空是否能正确触发胜利，盘面无合法交换是否能正确触发失败。
- **UI 文案验证**：确认目标文案、步数文案与实际规则一致。
