# PR2（基础交互与交换逻辑）

## 目标
完成棋子选中、相邻交换、无效回退和步数统计，让玩家能够完成基本交互。

## 任务 4：实现选中与取消选中
- **目标**：支持玩家点击棋子进行选中，并可取消。
- **实现内容**：
  - 点击棋子高亮
  - 再次点击已选中棋子取消选中
  - 点击空白区域取消当前选择
- **验收方式**：
  - 玩家能明确看到选中反馈
  - 可以正确取消选中状态

## 任务 5：实现相邻交换
- **目标**：支持玩家对上下左右相邻棋子进行交换。
- **实现内容**：
  - 判断相邻关系
  - 执行两个棋子位置交换
  - 播放交换动画
- **验收方式**：
  - 只能交换相邻元素
  - 非相邻元素无法交换
  - 交换动画表现正确

## 任务 6：实现无效交换回退
- **目标**：交换后若未形成消除，则回退到原位置。
- **实现内容**：
  - 交换后检测是否形成有效消除
  - 若无消除则自动回弹
  - 无效交换不计步数
- **实现假设**：
  - 按文档规则，普通交换若未形成可消除组合，视为无效操作并回退
- **验收方式**：
  - 无效交换会回退
  - 回退过程动画正常
  - 不会误触发后续流程

## 任务 7：实现步数统计
- **目标**：统计有效交换次数。
- **实现内容**：
  - 仅有效交换计步
  - 无效交换不计步
  - 结算界面显示总步数
- **验收方式**：
  - 每次有效交换步数 +1
  - 重置后步数清零

## PR2 联调验证
- **交换链路验证**：检查选中、交换、无效回退和步数更新是否形成闭环。
- **输入锁定初步验证**：交换过程中不允许重复触发第二次交换。
- **交互反馈验证**：选中、高亮、回弹表现符合预期。
